Это руководство охватывает установку сервера Ophix с нуля. Каждый сервер — это автономный экземпляр Django: пакет домена pip, установленный поверх `ophix-server-base`.

---

## Предварительные требования

- Python 3.10+
- MariaDB (движок по умолчанию) или другой поддерживаемый движок
- Веб-сервер (рекомендуется nginx)
- Доступ root или sudo для установки службы

---

## Пошаговая установка

Рекомендуемая последовательность использует две команды управления:

### Шаг 1 — Настройка установки

```bash
ophix-manage configure_install credserver
```

Интерактивный мастер запрашивает:

- Каталог установки
- Имя хоста и TLS-сертификаты (с проверкой CN/SAN)
- Данные для подключения к базе данных (проверка соединения в реальном времени)
- Данные суперпользователя
- Активация темы и заголовок панели администратора

Результат записывается в `.<slug>.conf` и `.env` (права доступа 600).

### Шаг 2 — Выполнение установки

```bash
ophix-manage run_install credserver
```

Эта команда:

- Создаёт структуру каталогов в `INSTALL_DIR`
- Копирует TLS-файлы в `ssl/certs/` и `ssl/private/`
- Генерирует конфигурационные файлы nginx и systemd
- Выполняет `migrate`, `collectstatic --noinput`
- Создаёт или обновляет суперпользователя
- Активирует тему и задаёт заголовок панели администратора
- Автоматически импортирует встроенную документацию

### Шаг 3 — Установка системной службы

```bash
sudo bash credserver_sudo_install.sh
```

Настраивает права доступа, устанавливает конфигурацию nginx, устанавливает и запускает службу systemd.

---

## Обновление

Для планового обновления:

```bash
pip install --upgrade ophix-server-base ophix-creds
ophix-manage migrate
ophix-manage collectstatic --noinput
sudo systemctl restart credserver
```

Если в новой версии добавлены переменные `.env`, сначала выполните `generate_deploy_config --append`.

---

## Среда разработки

Для локальной среды разработки:

```bash
python -m venv venv
source venv/bin/activate
pip install ophix-server-base ophix-creds ophix-docs
ophix-manage configure_install credserver
ophix-manage run_install credserver
ophix-manage runserver
```

## Справочник настроек

### Идентификация сервера

| Переменная | По умолчанию | Описание |
| --- | --- | --- |
| `SERVER_NAME` | _(обязательно)_ | Отображаемое имя этого экземпляра сервера |
| `SERVER_VERSION` | _(обязательно)_ | Версия развёрнутого серверного программного обеспечения |
| `INSTALL_DIR` | _(обязательно)_ | Корневой каталог установки |
| `ALLOWED_HOSTS` | _(обязательно)_ | Разрешённые имена хостов (через запятую) |
| `DEBUG` | `False` | Режим отладки Django. Никогда не включать в рабочей среде |

### Безопасность

| Переменная | По умолчанию | Описание |
| --- | --- | --- |
| `SECRET_KEY` | _(обязательно)_ | Секретный ключ Django. Генерируется автоматически при установке |
| `AUTH_LEAK_INFO` | `False` | Включать детали ошибок в ответы API. Только `True` для разработки |
| `MINIMUM_TOKEN_ROTATE_TIME` | `3600` | Минимальный интервал в секундах между ротациями токена |

### База данных

| Переменная | По умолчанию | Описание |
| --- | --- | --- |
| `DB_ENGINE` | `mariadb` | Движок базы данных. Варианты: `mariadb`, `postgres`, `mssql`, `oracle`, `cockroachdb` |
| `DB_NAME` | _(обязательно)_ | Имя базы данных |
| `DB_USER` | _(обязательно)_ | Пользователь базы данных |
| `DB_PASSWORD` | _(обязательно)_ | Пароль базы данных |
| `DB_HOST` | `localhost` | Хост базы данных |
| `DB_PORT` | _(по умолчанию движка)_ | Порт базы данных |

### Локализация

| Переменная | По умолчанию | Описание |
| --- | --- | --- |
| `LANGUAGE_CODE` | `en-us` | Код языка Django (например, `fr`, `de`, `ru`) |
| `TIME_ZONE` | `UTC` | Часовой пояс сервера |

### Административная панель

| Переменная | По умолчанию | Описание |
| --- | --- | --- |
| `ADMIN_SITE_HEADER` | `Ophix` | Заголовок панели администратора |
| `SHOW_CLIENT_ARTIFACT_MODEL` | `False` | Показывать модель связи клиент-артефакт в панели администратора |
| `SHOW_ACCESS_LOGS` | `False` | Показывать раздел журналов доступа в панели администратора |
| `SHOW_AUTH_MODELS` | `False` | Показывать модели аутентификации Django в панели администратора |
| `SHOW_THEME_MODEL` | `False` | Показывать модель темы в панели администратора |

### Журнал аудита

| Переменная | По умолчанию | Описание |
| --- | --- | --- |
| `AUDIT_BATCH_SIZE` | `50` | Количество событий в пакете записи |
| `AUDIT_FLUSH_INTERVAL` | `5` | Максимальная задержка сброса в секундах |

### Рекомендации для рабочей среды

- Всегда развёртывайте за обратным прокси (nginx)
- Прокси должен надёжно устанавливать `X-Forwarded-For` — аутентификация Ophix зависит от исходного IP-адреса
- В рабочей среде всегда сохраняйте `DEBUG=False`
- Резервную копию файла `.env` храните отдельно от базы данных — он содержит ключи шифрования
